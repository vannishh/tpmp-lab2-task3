# Отчет по лабораторной работе - Задание 3

## Цель работы
Разработать программу на языке C с использованием структур для управления данными о студентах и их успеваемости. Реализовать все необходимые функции согласно варианту задания. Настроить сборку проекта с помощью make и GitHub Actions. Добавить репозиторий как submodule в основную лабораторную работу.

## Описание структуры данных
Структура STUDENT содержит следующие поля:
- `surname` - фамилия студента
- `initials` - инициалы студента  
- `group` - номер группы
- `grades[4]` - массив из четырех оценок (5-балльная система)

## Реализованные функции

### 1. create_students()
Создает массив записей о студентах. Осуществляет ввод данных с клавиатуры для 7 студентов.

### 2. calculate_averages()
Вычисляет средний балл для каждого студента на основе четырех оценок.

### 3. sort_by_average()
Сортирует записи о студентах по убыванию среднего балла.

### 4. print_excellent_students()
Выводит сведения о студентах, имеющих только оценки 4 и 5.

### 5. remove_min_average_student()
Удаляет из списка студента с минимальным средним баллом.

### 6. print_all_students()
Вспомогательная функция для вывода всех студентов.

## Модульная структура проекта (КИС-2)

### Клиентская часть
- **main.c**: главный модуль, содержащий функцию main() и демонстрирующий работу всех функций

### Интерфейсная часть
- **student.h**: определение структуры STUDENT и констант
- **functions.h**: прототипы всех функций для работы со студентами

### Серверная часть
- **functions.c**: реализация всех функций обработки данных о студентах

## Проверка сборки и выполнения

### Локальная сборка
```bash
$ make
gcc -Wall -Wextra -std=c99 -c src/main.c -o build/main.o
gcc -Wall -Wextra -std=c99 -c src/functions.c -o build/functions.o
gcc -Wall -Wextra -std=c99 build/main.o build/functions.o -o student_program
```

### Запуск программы
```bash
$ ./student_program
=== Программа учета успеваемости студентов ===
Ввод данных для 7 студентов:
...
```

### GitHub Actions
При каждом push в репозиторий автоматически выполняется:
1. Проверка кода
2. Сборка проекта с помощью make
3. Запуск программы
